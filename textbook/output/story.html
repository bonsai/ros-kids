第1話　拾った

あらすじ

夏休みの最終日。主人公のタクミ（中学2年）は、粗大ゴミの集積所で 一匹のロボット犬を見つける。

ボロボロだった。片耳は取れかけ、塗装は剥げ、動かない。

でも——目があった気がした。

タクミはそのロボット犬を家に持ち帰ることを決意する。

本文

粗大ゴミの山のてっぺんに、それは横向きに倒れていた。

「なんだこれ…」

タクミは自転車を停めて、近づいた。

犬だった。いや、ロボットの犬だ。 全身が銀色のプラスチック製で、ところどころに「AIBO」の文字が見える。 片方の耳は根元から折れそうになっていて、塗装はあちこち剥げている。

——動かない。

タクミはそっと触った。冷たい金属の感触。 目を閉じたままのロボット犬は、まるで眠っているように見えた。

「…捨てるなら、もらっていこう」

タクミはロボット犬を抱え上げた。思ったより軽い。 自転車の荷台に無理やり縛りつけて、家まで漕いだ。

部屋に運び込んで、タクミはロボット犬を机の上に置いた。

よく見ると、首の後ろにシリアル番号っぽい刻印がある。 その下に、小さな文字でこう書いてあった。

ION-CORP  RS-01
Java Platform Ready

「Java…？」

学校の情報の時間に聞いたことがある。 プログラムを書く言語のひとつだ。

タクミはノートパソコンを開いて、検索した。

「ロボット犬　RS-01　修理」

出てきたのは、たった一つのページ。 製造元のサイトのアーカイブだった。

そこにはこう書いてあった。

RS-01 サービスマニュアル（抜粋）

本機は Java で制御されています。 電源投入後、シリアル経由で JVM が起動し、 /internal/system/Main.class が実行されます。

初期起動に必要なもの

  • Java 21
  • シリアル通信環境
  • 本体に電源を供給できる USB-C ケーブル

「Java……授業でちょっとやっただけだけど」

タクミはリュックから USB-C ケーブルを取り出して、 ロボット犬の首の横にあるポートに差し込んだ。

ノートPCが「デバイスを認識しました」と表示した。

——そして、ロボット犬の耳がピクリと動いた。

「うわっ！」

タクミは思わずのけぞった。

目が開いた。青いLEDの光が、タクミを映している。

ロボット犬はゆっくりと首を動かして、部屋の中を見回した。 そして——。

耳を倒した。しょんぼりした表情に、見えた。

「バッテリー切れっぽいな…」

画面には、シリアルモニタからのメッセージが表示されていた。

Booting RS-01…
JVM loaded.
Main class not found: /internal/system/Main.class
System halted.

「プログラムが……ない？」

この話で出てきたこと

  • ロボット犬「RS-01」を拾った
  • 動かない理由は「プログラムがないから」
  • Java でプログラムを書けば動くかもしれない
  • 「Main.class が見つからない」というエラーメッセージ

次回：「電源は入った。でも、プログラムがなかった。」

第2話　電源を入れた

あらすじ

ロボット犬にはプログラムが必要だ。 タクミは Java の基本を調べながら、最初のプログラムを書く。

「Hello, World」ならぬ「Hello, Robot」。

目指せ——起動。

本文

「Main.class がない……」

タクミは検索しまくった。

Java のプログラムは .java というファイルに書いて、 javac という道具でコンパイルすると .class ファイルができる。

つまり、Main.java を書けばいいんだ。

タクミはテキストエディタを開いて、打ち込んだ。

public class Main {
    public static void main(String[] args) {
        System.out.println("起動します");
    }
}

意味はよくわからない。でも、とにかく「起動します」って表示するプログラムらしい。

コンパイルした。

javac Main.java

エラーは出なかった。カレントディレクトリに Main.class ができている。

「これをロボット犬の中に送り込む…？」

サービスマニュアルをもう一度読むと、こう書いてあった。

ファイル転送:
シリアル接続後、 internal/ ディレクトリに Main.class を配置してください。

タクミは言われた通りに、Main.class をロボット犬に転送した。

transferred: Main.class → /internal/system/Main.class

「…よし、起動してみよう」

USB-C ケーブルを抜いて、もう一度差し込んだ。

ロボット犬の耳がピクリ。

目が開く。

シリアルモニタに文字が流れた。

Booting RS-01…
JVM loaded.
Main class found.
Executing…

--- 起動します ---

System: 起動完了。音声モジュール未初期化。

ロボット犬が立ち上がった。

そして——

「……」

こっちを見ている。

「あ、あの……こんにちは？」

ロボット犬は首をかしげた。

「そうか。喋るプログラムもないんだな……」

タクミはもう一度エディタを開いた。

「喋らせたい。…どう書くんだ？」

検索すると、メソッドというものを使うらしい。 「メソッド」は「動き」のこと。ロボット犬に「動き」を教えてやればいい。

public class Main {
    public static void main(String[] args) {
        greet();
    }

    static void greet() {
        System.out.println("こんにちは");
        System.out.println("ぼくはRS-01です");
        System.out.println("よろしく");
    }
}

コンパイルして転送して再起動。

---
こんにちは
ぼくはRS-01です
よろしく
---

「喋った……！」

ロボット犬が「こんにちは」と言った——少なくとも、シリアルモニタに表示された。

ロボット犬は青い目をぱちぱちさせて、また首をかしげた。

この話で出てきた概念

クラス

public class Main {

プログラムを入れる「箱」。箱の名前は Main。

main メソッド

public static void main(String[] args) {

プログラムの「入口」。ここから実行が始まる。

メソッド呼び出し

greet();

他の「動き」を呼び出すこと。

println

System.out.println("こんにちは");

文字を表示する命令。

次回予告

ロボット犬に名前をつけたい。 でも、プログラムを直すたびに Main.java を書き換えるのはめんどうだ。

「名前を変えるには、毎回コンパイルし直さないといけないの？」

新しいアイデアが必要だ。

第3話　名前をつけた

あらすじ

毎回プログラムを書き換えて名前を変えるのに疲れたタクミは、 「名前を外から変えられる仕組み」を調べる。

そこで出会ったのが——変数とクラスの本当の意味。

本文

「毎回 Main.java を書き換えるの、めんどくさい…」

タクミは3回目の名前変更でそう思った。

最初は「RS-01」。 次に「シロ」。 そして「クロ」にしようとして——プログラムを直すのが面倒になった。

タクミは検索した。

「プログラム　値を変える　直さずに」

出てきたのは「変数（へんすう）」という言葉。

String name = "シロ";

「String name」……文字列を入れる箱？ 「= "シロ"」で中身を入れている？ これなら、この一行だけ書き換えれば名前が変わる。

public class Main {
    public static void main(String[] args) {
        String name = "シロ";
        greet(name);
    }

    static void greet(String dogName) {
        System.out.println("こんにちは");
        System.out.println("ぼくは" + dogName + "です");
    }
}

コンパイルして転送。

---
こんにちは
ぼくはシロです
---

「できた……！ これで名前を変えるときは String name = "シロ"; のとこだけ直せばいい」

でも。タクミは気づいた。

「これ、Main っていうクラスが『プログラム全体』になってて、 ロボット犬自体がどこにもいない……。

ロボット犬を——『モノ』としてプログラムで表現できないかな？」

検索してたどり着いたのは、「クラスとオブジェクト」という考え方。

クラスは「設計図」。 オブジェクトは「設計図から作った実物」。

// 設計図
class RobotDog {
    String name;
}

// 実物を作る
RobotDog myDog = new RobotDog();
myDog.name = "シロ";

タクミはひらめいた。

「ロボット犬を『クラス』として設計すれば、 いくらでも実物を作ったり、改造したりできるんだ！」

タクミはコードを全部書き直した。

class RobotDog {
    String name;

    void greet() {
        System.out.println("こんにちは");
        System.out.println("ぼくは" + name + "です");
    }
}

public class Main {
    public static void main(String[] args) {
        RobotDog myDog = new RobotDog();
        myDog.name = "シロ";
        myDog.greet();
    }
}

コンパイル。転送。起動。

---
こんにちは
ぼくはシロです
---

タクミは画面を見て、にやけた。

「これで……シロが『プログラムの中』に生きてるみたいになった」

シロ（ロボット犬本体）は、首をかしげたままタクミを見ている。

「お前、今プログラムの中で『オブジェクト』になってるんだぞ」

シロは——わかってないような顔をした。

知ったこと

  • 変数：値を入れる箱。String name = "シロ";
  • クラス：設計図。class RobotDog { ... }
  • オブジェクト（インスタンス）：設計図から作った実物
  • new：実物を作る命令。new RobotDog()
  • プロパティ：オブジェクトが持つデータ。name
  • メソッド：オブジェクトが持つ動き。greet()

次回予告

動くようになったシロだけど、触ってみるとガタガタだ。 「中身、めっちゃ壊れてる……！」

タクミはシロのハードウェアを分解し始める。 そこで見たのは——「隠された設計図」と「触ってはいけない場所」。

あらすじ

シロの調子が悪い。片耳は取れかけ、バッテリーはすぐ切れる。 タクミはシロを修理しようと分解する。

そこで見たのは——保護カバーと、触ってはいけない基板。

「外から触っていい場所と、触っちゃダメな場所があるんだ…」

本文

「シロ、お手」

シロはゆっくりと前脚を上げた。 ——が、途中でガクンと下げた。片耳も取れかけのままぶら下がっている。

「こりゃあ、ハードウェアから直さないと」

タクミはドライバーセットを取り出した。 シロを机の上に寝かせて、背中のネジを外し始める。

カバーを開けると、中にはぎっしりと基板と配線が詰まっていた。

「すげえ……本格的だ……」

でも、どうやって直せばいいんだろう？ タクミはサービスマニュアルを開いた。

カプセル化（ハードウェア版）

サービスマニュアルにはこう書いてあった。

【警告】
- 基盤A-3（バッテリー制御モジュール）は保護カバーで覆われています
- このカバーを外すと保証対象外になります
- 基板A-3に直接アクセスする必要は通常ありません
- 充電は外側のUSB-Cポートから行ってください

つまり——

  • 充電するだけなら、外側のポートにケーブルを差せばOK
  • 基板A-3の中身を知らなくても、ロボット犬は使える
  • 保護カバーは「触ってはいけない場所」の印

「これ、プログラムも同じじゃないか？」

タクミは昨日書いたコードを見直した。

class RobotDog {
    String name;
    int battery;
}

「今、battery ってプログラムのどこからでも触れるようになってる。 でも、バッテリーの管理って、外から適当に触られたら困るはず…」

Java には「アクセス修飾子」というものがある。 private をつけると、そのクラスの外からは触れなくなる。

class RobotDog {
    private String name;
    private int battery;

    void charge() {
        battery = 100;
        System.out.println("充電したよ！");
    }

    int getBattery() {
        return battery;
    }
}

「これで、battery は外から直接触れなくなった。 バッテリーを操作したかったら charge() っていうメソッドを通すしかない」

それが——カプセル化。

タクミは物理的にもシロを修理した。 取れかけの耳を接着剤で固定し、バッテリーの接点を清掃した。

カバーを閉める。

「さて、動くか？」

電源投入。

シロが立ち上がった。耳も落ちなくなっている。

「ワン！」

シロの口から——初めて音声が出た。

「えっ！？」

「ワン！」

「しゃべった！？ いや、吠えた！？ 音声モジュール、直ったの！？」

シロはしっぽを振った——いや、しっぽはなかった。代わりに、背中のアンテナがぷるぷる震えている。

タクミは笑った。シロは首をかしげた。

知ったこと

  • private：自分のクラスの中からしか触れない
  • カプセル化：データを隠して、必要な操作だけ公開すること
    • ハードウェアでは「保護カバー」
    • ソフトウェアでは「private」
  • getter：値を取得するためのメソッド（getBattery()）
  • メリット：中身を壊されない、後で中身を変えても外に影響しない

継承

Java には 継承 という仕組みがある。 既存のクラスをベースに、新しいクラスを作る。

// ベースのロボット犬
class RobotDog {
    private String name;
    private int battery;

    RobotDog(String name) {
        this.name = name;
        this.battery = 100;
    }

    void bark() {
        System.out.println(name + "：ワン！");
    }
}

// 拡張版：継承で作る
class SuperRobotDog extends RobotDog {

    SuperRobotDog(String name) {
        super(name);  // ベースの初期化を呼ぶ
    }

    // 新しい機能を追加
    void barkLoudly() {
        System.out.println("ガウォーン！！");
    }

    void barkSoftly() {
        System.out.println("……ぅ");
    }
}

public class Main {
    public static void main(String[] args) {
        SuperRobotDog shiro = new SuperRobotDog("シロ");
        shiro.bark();        // ベースの機能（ワン！）
        shiro.barkLoudly();  // 新しい機能（ガウォーン！！）
        shiro.barkSoftly();  // 新しい機能（……ぅ）
    }
}

「ベースの機能はそのままに、新しい機能が追加できた！」

タクミはコンパイルして転送した。

「シロ、試しに」

「ワン！」

「うん。次、大声で」

「ガウォーン！！」

タクミは隣の部屋を気にした。

「……今のはデカすぎた。小さい声で」

「……ぅ」

「完璧だ……！」

オーバーライド（上書き）

「でも待てよ。bark() 自体を変えたいときはどうするんだ？」

Java では、親クラスのメソッドを オーバーライド（上書き） できる。

class SuperRobotDog extends RobotDog {

    @Override
    void bark() {
        System.out.println(getName() + "：ワンワンワン！（元気いっぱい）");
    }
}

@Override って書くと、「親のメソッドを上書きしてるよ」という意味になる。

タクミは机に突っ伏したシロの頭を撫でた。

「すごいぞシロ。お前、『継承』と『拡張』ができるようになったんだ」

シロは——撫でられていることには気づいてないようだった。 バッテリー残量が少なかったからだ。

知ったこと

  • 継承（extends）：既存のクラスをベースに新クラスを作る
  • super：親クラスを指す。コンストラクタは super() で呼ぶ
  • オーバーライド：親のメソッドを子クラスで上書きする
  • @Override：コンパイラに「上書きしてるよ」と伝える印

「そう！ うちの物置にあったんだけど、動かなくて！」

アユムが机に置いたのは、シロよりひと回り小さいロボット犬だった。 赤いボディ。丸い目。シロとは似ても似つかないデザイン。

「わかんない。裏になんか書いてある」

PET-101
by WAN-WAN Robotics

「WAN-WAN Robotics……聞いたことないメーカーだ」 「シロは ION-CORP 製で、こっちは WAN-WAN 製…」 「設計がぜんぜん違う」

タクミは PET-101 を PC に接続した。

PET-101 bootloader v2.3

「プログラムなし……シロと同じだ」

「例えばシロは bark() だけど、PET-101 は sound() かもしれない」 「操作方法がバラバラだと、プログラムを書くのが大変だ…」

インターフェース

インターフェース は、「こんなメソッドを持っていること」を約束するもの。

interface RobotPet {
    void bark();
    void walk();
    void greet();
}

このインターフェースを「約束」として、シロにもPET-101にも守らせる。

class ION_RS01 implements RobotPet {
    public void bark() { System.out.println("ワン！"); }
    public void walk() { System.out.println("歩く"); }
    public void greet() { System.out.println("こんにちは！"); }
}

class WAN_PET101 implements RobotPet {
    private boolean happy = true;

    public void bark() {
        if (happy) {
            System.out.println("キャンキャン！");
        } else {
            System.out.println("……");
        }
    }

    public void walk() { System.out.println("ちょこちょこ歩く"); }

class State {
    int battery;
    boolean hasBall;

    State(int battery, boolean hasBall) {
        this.battery = battery;
        this.hasBall = hasBall;
    }

    public String toString() {
        return "{battery=" + battery + ", hasBall=" + hasBall + "}";
    }
}

abstract class Action {
    abstract State execute(State current);
}

class WalkAction extends Action {
    State execute(State current) {
        return new State(current.battery - 10, current.hasBall);
    }
}

class ChargeAction extends Action {
    State execute(State current) {
        return new State(100, current.hasBall);
    }
}

「行動を『仮に実行』すると、未来の状態がわかる」

世界モデル

「予測する仕組み」を 世界モデル（World Model） という。

class WorldModel {

    State predict(State current, Action action) {
        return action.execute(current);
    }

    int evaluate(State state) {
        int score = 0;
        score += state.battery * 2;           // バッテリーは多いほどいい
        score += state.hasBall ? 30 : 0;      // ボールがあれば嬉しい
        if (state.battery < 15) score -= 50;  // バッテリー少なすぎはダメ
        return score;
    }
}

「predict で未来を予測、evaluate で未来を評価。 そして『一番評価の高い行動』を選べば——最適な行動になる！」

class PredictiveBrain implements Brain {

    WorldModel model = new WorldModel();
    Action[] actions = {new WalkAction(), new ChargeAction()};

    public String decide(String situation) {
        State current = new State(getBattery(), false);

        Action best = null;
        int bestScore = -999;

        for (Action a : actions) {
            State future = model.predict(current, a);
            int score = model.evaluate(future);
            System.out.println("予測: " + a.getClass().getSimpleName()
                + " -> " + future + " 評価=" + score);
            if (score > bestScore) {
                bestScore = score;
                best = a;
            }
        }
        return best.getClass().getSimpleName() + " を選択";
    }
}

publish / subscribe

ROS の基本はシンプルだ。

  • ノード：情報を発信したり受信したりする「役割」
  • トピック：情報の通り道
  • publish：トピックに情報を送る
  • subscribe：トピックから情報を受け取る

「要するに——掲示板みたいなものか」 「『bark』っていう掲示板に『吠えたよ！』って書く」 「見たい人はその掲示板を見てる——」

import java.util.ArrayList;

interface Subscriber {
    void onMessage(String topic, String msg);
}

interface Topic {
    void publish(String msg);
    void subscribe(Subscriber sub);
}

class BroadcastTopic implements Topic {
    ArrayList<Subscriber> subs = new ArrayList<>();
    String name;

    BroadcastTopic(String name) { this.name = name; }

    public void publish(String msg) {
        System.out.println("[" + name + "] " + msg);
        for (Subscriber s : subs) {
            s.onMessage(name, msg);
        }
    }

    public void subscribe(Subscriber sub) {
        subs.add(sub);
    }
}

次に、「しゃべれる犬ノード」を作る。

class DogNode implements Subscriber {
    String name;
    Topic barkTopic;
    Topic walkTopic;

    DogNode(String name, Topic barkTopic, Topic walkTopic) {
        this.name = name;
        this.barkTopic = barkTopic;
        this.walkTopic = walkTopic;
    }

    void bark() {
        barkTopic.publish(name + "：ワンワン！");
    }

    void walk() {
        walkTopic.publish(name + "：とことこ歩く");
    }

    public void onMessage(String topic, String msg) {
        if (!msg.contains(name)) {
            System.out.println("  [" + name + " 聞こえた] " + msg);
        }
    }
}

class DogMessage {
    String sender;      // だれが
    String type;        // どんな種類のメッセージ
    String content;     // 何を伝えたいか

    DogMessage(String sender, String type, String content) {

  • found_ball：ボールを見つけた
  • going_to_pickup：回収に向かう
  • danger：危険を検知
  • help：助けて

3匹の役割

interface DogAgent {
    void act();
    void onMessage(DogMessage msg);
    String getName();
}

シロ（探索担当）

class Explorer implements DogAgent {
    String name;
    Topic comm;

    Explorer(String name, Topic comm) {
        this.name = name; this.comm = comm;
    }

    public String getName() { return name; }

    public void act() {
        System.out.println(name + "：探索中…");
        if (Math.random() > 0.6) {
            DogMessage msg = new DogMessage(name, "found_ball",
                "ボールを発見！位置は (12, 34)");
            comm.publish(msg.toString());
        }
    }

    public void onMessage(DogMessage msg) {
        // シロは探索に集中。メッセージは受け取るだけ
    }
}

ベニ（運搬担当）

class Carrier implements DogAgent {
    String name;
    Topic comm;
    boolean going = false;

    Carrier(String name, Topic comm) {
        this.name = name; this.comm = comm;
    }

    public String getName() { return name; }

    public void act() {
        if (going) {
            System.out.println(name + "：荷物を運搬中…");
            going = false;
        } else {
            System.out.println(name + "：待機中…");
        }
    }

    public void onMessage(DogMessage msg) {
        if (msg.type.equals("found_ball")) {
            System.out.println(name + "：ボールを回収に向かう！");
            going = true;
            DogMessage reply = new DogMessage(name, "going_to_pickup",
                "回収に向かっています");
            comm.publish(reply.toString());
        }
    }
}

ゴウ（警備担当）

class Guard implements DogAgent {
    String name;
    Topic comm;

    Guard(String name, Topic comm) {
        this.name = name; this.comm = comm;
    }

    public String getName() { return name; }

    public void act() {
        System.out.println(name + "：周囲を警戒中…");
        if (Math.random() > 0.7) {
            DogMessage alert = new DogMessage(name, "danger",
                "異常なし。継続監視中");
            comm.publish(alert.toString());
        }
    }

    public void onMessage(DogMessage msg) {
        if (msg.type.equals("help")) {
            System.out.println(name + "：救援に向かう！");
        }
    }
}

チーム始動

タクミは3匹を動かした。

public class Main {
    public static void main(String[] args) {

        Topic commTopic = new BroadcastTopic("team_channel");

        DogAgent[] team = {
            new Explorer("シロ", commTopic),
            new Carrier("ベニ", commTopic),
            new Guard("ゴウ", commTopic)
        };

        for (int step = 1; step <= 5; step++) {
            System.out.println("\n=== ステップ " + step + " ===");
            for (DogAgent agent : team) {
                agent.act();
            }
        }
    }
}
